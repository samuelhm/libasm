Registros de Estado y Control
Registro Descripción ¿Para qué sirve?
RIP
EIP
IP
 Puntero de Instrucción. Apunta a la siguiente instrucción que la CPU va a ejecutar. Modificado por CALL, RET, JMP y saltos condicionales. No se puede acceder directamente como operando (se lee con LEA o CALL). Controla el flujo de ejecución. La CPU lee RIP para saber qué instrucción ejecutar a continuación. Es la brújula del programa.
RFLAGS
EFLAGS
FLAGS
CF PF AF ZF SF TF IF DF OF IOPL NT RF VM AC ID
Contiene las banderas de estado (CF, ZF, SF, OF, PF, AF) y de control (IF, DF, TF). Modificado por operaciones aritméticas, lógicas y de comparación. Las banderas determinan el flujo de los saltos condicionales. Define el resultado de comparaciones y operaciones. Los saltos condicionales (je, jne, jg, jl…) consultan estas banderas para decidir si saltar o no. Sin RFLAGS no hay if ni bucles.

Registros de Control del Sistema
Estos registros solo pueden ser leídos/escritos desde anillo 0 (kernel). Un programa en espacio de usuario que intente acceder a ellos (p. ej. MOV CR0, RAX) provocará una excepción #GP(0) (General Protection Fault).
Registro Propósito
CR0 Control del modo de operación del procesador. Flags: PE (Protected Mode), MP, EM, TS, ET, NE, WP, AM, NW, CD, PG (Paging).
CR2 Contiene la dirección lineal que causó un Page Fault. El kernel lo lee en el handler de #PF para saber qué dirección falló.
CR3 Puntero a la Page Table de nivel superior (PML4 en x86_64). Se recarga en cada cambio de contexto para cambiar el espacio de memoria virtual.
CR4 Habilita extensiones arquitectónicas: PAE, PSE, PGE, OSFXSR, OSXMMEXCPT, UMIP, SMEP, SMAP, etc.
CR8 Task Priority Register. Controla la prioridad de interrupción que acepta el procesador (solo en modo x86_64).
